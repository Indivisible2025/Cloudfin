# Cloudfin UI — 桌面端（Windows / Linux / macOS）

> 版本：v0.1.0  
> 日期：2026-04-03  
> 状态：**草案 — 待审查**

---

## 1. 概述

桌面端 UI 是独立于 Core 的原生应用，通过 WebSocket 与 Cloudfin Core 守护进程通信。面向**多任务、高密度信息**场景，强调**效率和专业感**。

**技术栈**：
- Windows：C# + WinUI 3 或 C++ + Qt
- macOS：Swift + SwiftUI
- Linux：Rust (QML) / GTK / Qt

> **跨平台说明**：Linux / macOS / Windows 三平台的 UI 实现尚未完成，但架构与逻辑相同。完成后各平台的交互体验保持一致。

**通信架构**：
```
┌──────────────────────────────────┐
│        Cloudfin UI (桌面应用)       │
│  (WinUI3 / SwiftUI / Qt / GTK)   │
└──────────────┬─────────────────────┘
               │ WebSocket (127.0.0.1:19001/ws)
               ▼
┌──────────────────────────────────┐
│        Cloudfin Core (守护进程)      │
│          127.0.0.1:19001          │
└──────────────┬─────────────────────┘
               │ .so / .dylib / .dll
               ▼
┌──────────────────────────────────┐
│           cloudfin-mod/            │
│    p2p / crdt / tor / i2p / storage│
└──────────────────────────────────┘
```

## 2. 窗口布局

### 2.1 主窗口结构

**单窗口 + 侧边栏布局**，充分利用大屏空间：

```
┌─────────────────────────────────────────────────────────────┐
│ [≡]  ☁ 云鳞 Cloudfin          [搜索...]   [_][□][×]      │  ← 标题栏
├──────────┬────────────────────────────────────────────────┤
│          │                                                │
│ 核心     │                                                │
│ 通信     │          主内容区                               │
│ 加密     │      （根据左侧选中项切换）                      │
│ 同步     │                                                │
│ 设置     │                                                │
│          │                                                │
├──────────┼────────────────────────────────────────────────┤
│          │ Core: ● 运行中   通信: ●   加密: ○   同步: ●    │  ← 底部状态栏
└──────────┴────────────────────────────────────────────────┘
   220px                    剩余空间
```

**侧边栏**：固定宽度 220px，图标 + 文字标签，垂直排列

| 入口 | 说明 |
|------|------|
| 核心 | Core 自身状态总览 |
| 通信 | 通信类模块（P2P、TOR、I2P 等）状态与配置 |
| 加密 | 加密类模块状态与配置 |
| 同步 | 同步类模块（CRDT）状态与配置 |
| 设置 | 应用设置 |

**主内容区**：根据选中项显示对应页面，支持多标签页

**状态栏**：固定在底部，显示 Core 状态 + 各分类模块数量

## 6.2 图片背景图层架构

**核心原则**：背景层与内容层完全分离，背景固定不动，内容层覆盖其上。

**完整图层栈（底→顶）**：

```
┌──────────────────────────────────────────────────────┐  ← 图层0：根容器（Stack/Overlay）
│                                                      │
│  ┌──────────────────────────────────────────────┐  │
│  │ 图层1: Image.asset(图片)                    │  │  ← 充满整个窗口，无外边距
│  │ Window.SetBackgroundImage() 或 Stack 内 Image  │  │  ← 固定不滚动
│  └──────────────────────────────────────────────┘  │
│                                                      │
│  ┌──────────────────────────────────────────────┐  │
│  │ 图层2: 遮罩层                               │  │  ← Container(color)
│  │ Color.fromRGBO(0,0,0,遮罩强度)             │  │    alpha = 遮罩强度 × 255
│  └──────────────────────────────────────────────┘  │    范围: 0.40 ~ 0.70
│                                                      │
│  ┌──────────────────────────────────────────────┐  │
│  │ 图层3: 内容层                                │  │  ← 主应用内容区域
│  │ ┌──────────────────────────────────────────┐ │  │
│  │ │ 侧边栏  │  主内容区（固定背景）          │ │  │
│  │ │ 透明背景│  透明背景，内容区无独立背景    │ │  │
│  │ └──────────────────────────────────────────┘ │  │
│  │ 内容区背景: transparent                       │  │
│  │ 侧边栏背景: Color.fromRGBO(0,0,0,0.60)      │  │
│  │ 卡片背景: transparent                        │  │
│  │ 卡片边框: BorderSide(color:white.alpha(10))  │  │
│  └──────────────────────────────────────────────┘  │
│                                                      │
│  ┌──────────────────────────────────────────────┐  │
│  │ 图层4: 状态栏                                │  │  ← 底部固定状态栏
│  │ Container(color: Color.fromRGBO(0,0,0,0.70)) │  │  ← 自身保证可读性
│  └──────────────────────────────────────────────┘  │
│                                                      │
│  ┌──────────────────────────────────────────────┐  │
│  │ 图层5: Dialog / 悬浮面板                     │  │  ← 自带深色半透明背景
│  │ Container(color: Color.fromRGBO(30,30,30,0.95))│ │  ← 禁止继承透明父背景
│  └──────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────┘
```

**遮罩强度预设**：
| 强度值 | 适用场景 |
|--------|---------|
| 0.40 | 极浅/模糊壁纸 |
| 0.50 | 默认推荐 |
| 0.60 | 中等复杂度壁纸 |
| 0.70 | 高饱和/复杂壁纸 |

### 6.3 图片背景组件规范

**桌面端窗口背景实现**：
```dart
// WinUI3 / WPF
window.SetBackground(new ImageBrush { ImageSource = wallpaper, Stretch = Stretch.UniformToFill });

// Qt / QML
ApplicationWindow {
  background: Image {
    source: wallpaperPath
    fillMode: Image.PreserveAspectCrop
    anchors.fill: parent
  }
}

// SwiftUI
.background(
  Image(wallpaperName)
    .resizable()
    .aspectRatio(contentMode: .fill)
    .frame(maxWidth: .infinity, maxHeight: .infinity)
    .clipped()
)
```

**侧边栏特殊处理**：
- 侧边栏背景: `Color.fromRGBO(0, 0, 0, 0.55~0.65)` 固定深色
- 不依赖透明度继承，保证在任何壁纸上可读
- 与主内容区分隔明显

**窗口尺寸变化**：
```
✓ 图片使用 PreserveAspectCrop（可能裁剪）
✓ 禁止使用 Stretch = Fill / None
✓ 图片位置: 居中 (Alignment.Center)
✓ 窗口resize时背景图片不动（fixed attachment）
```

**对话框/弹窗**：
```
✓ 弹窗自带背景: Color.fromRGBO(30, 30, 30, 0.96)
✓ 禁止直接继承透明父容器背景
✓ 可选叠加毛玻璃: blur sigma=8
✓ 弹窗边框: BorderSide(color: white.alpha(15))
```

## 11. WebSocket 消息协议

桌面端 UI 通过 WebSocket 与 Cloudfin Core 守护进程通信，所有消息均为 JSON 格式。

**默认地址**：`ws://127.0.0.1:19001/ws`

### 11.1 连接与认证

Core 不需要额外认证（本地通信）。连接建立后，UI 应立即发送 `get_core_status` 请求确认 Core 可用性。

**连接状态定义**：

| 状态 | 说明 |
|------|------|
| `connecting` | 正在建立连接 |
| `connected` | 连接已建立，可收发消息 |
| `reconnecting` | 连接断开，正在重连 |
| `disconnected` | 连接关闭，未重连 |

**重连策略**：
- 首次连接失败：立即重试，间隔 1s、2s、4s、8s、16s（指数退避），最大间隔 30s
- 连接中收到关闭帧：等待 1s 后重连
- 最多重试 10 次，超此后提示用户检查 Core 状态

### 11.2 通用消息格式

**请求**：
```json
{
  "id": "req-uuid-001",
  "method": "get_core_status",
  "params": {}
}
```

**响应**：
```json
{
  "id": "req-uuid-001",
  "ok": true,
  "result": { ... }
}
```

**错误**：
```json
{
  "id": "req-uuid-001",
  "ok": false,
  "error": {
    "code": -32600,
    "message": "Module not found",
    "data": { "module_id": "unknown-module" }
  }
}
```

**推送（无 `id` 字段）**：
```json
{
  "method": "status_update",
  "params": { ... }
}
```

### 11.3 请求/响应详解

#### `get_core_status`

## 12. Cards 动态渲染机制

### 12.1 数据流

```
manifest.json → Core（解析 cards） → WS get_modules 响应 → UI 动态渲染
```

**模块 manifest.json**（简化示例）：
```json
{
  "id": "p2p",
  "name": "P2P",
  "version": "0.54",
  "category": "Network",
  "cards": {
    "info": [
      { "key": "listen_addr", "label": "监听地址", "style": "text" }
    ],
    "actions": [
      { "key": "refresh", "label": "刷新节点", "action": "refresh_nodes" }
    ],
    "settings": [
      { "key": "listen_addr", "label": "监听地址", "type": "text" }
    ]
  }
}
```

Core 启动时扫描模块目录，解析每个模块的 manifest.json，将 cards 数据缓存在内存中，通过 WS `get_modules` 响应返回给 UI。

### 12.2 侧边栏 4 类与模块 category 映射

| 侧边栏入口 | 对应 category | 示例模块 |
|-----------|---------------|---------|
| 核心 | `Core`（特殊标识，非模块） | Core 自身（非模块） |
| 通信 | `Network` | p2p, tor, i2p |
| 加密 | `Crypto` | crypto-box, openpgp |
| 同步 | `Sync` | crdt, dat |
| 设置 | 全局设置 + market | 非模块页 |

**category 枚举**：
- `Network` → 侧边栏"通信"
- `Crypto` → 侧边栏"加密"
- `Sync` → 侧边栏"同步"

### 12.3 动态卡片渲染器

UI 实现一个通用的 `CardsRenderer`，根据 cards 类型分发渲染：

#### info 卡片（纯展示）

```json
[
  { "key": "listen_addr", "label": "监听地址", "style": "text" },
  { "key": "kbucket_size", "label": "K-Bucket 大小", "style": "number" },
  { "key": "mdns_enabled", "label": "mDNS 发现", "style": "status" },
  { "key": "bytes_in", "label": "流入流量", "style": "number" }
]
```

`style` 可选值：`text` | `number` | `duration` | `status` | `progress`

- `text` / `number` → 标签 + 值（等宽字体显示值）
- `status` → 标签 + 状态色点（green/yellow/red）
- `duration` → 标签 + 时长格式化（如 `2h 34m`）
- `progress` → 标签 + 进度条（0–100）

#### actions 卡片（按钮触发 WS）

```json
[
  { "key": "refresh",       "label": "刷新节点",  "action": "refresh_nodes" },
  { "key": "add_bootstrap", "label": "添加节点",  "action": "add_bootstrap_node" },
  { "key": "clear_cache",   "label": "清理缓存",  "action": "clear_cache" }
]
```

**`action` 字段**：点击后触发的 WebSocket 方法名（如 `refresh_nodes`、`add_bootstrap_node`），由 Core 实现具体逻辑。

- 渲染为图标按钮或文字按钮
- 运行中模块的 actions 可直接点击；停止中模块的 actions 应禁用并提示"请先启动模块"

#### settings 卡片（表单编辑 + 保存触发 WS）

```json
[
  { "key": "listen_addr",  "label": "监听地址",   "type": "text",     "default": "/ip4/0.0.0.0/tcp/42221" },
  { "key": "enable_mdns",  "label": "启用 mDNS",  "type": "toggle",   "default": true },
  { "key": "kbucket_size", "label": "K-Bucket 大小", "type": "number", "default": 20 },
  { "key": "bootstrap",    "label": "Bootstrap 节点", "type": "select", "default": "..." }
]
```

`type` 可选值：`text` | `number` | `toggle` | `select` | `password`

- 渲染为表单控件（TextBox / Toggle / NumberBox / Select / PasswordBox）
- `default` 字段提供默认值（非必填，不填时使用空值）
- 表单底部固定「保存」「重置」按钮
- 点击「保存」→ 收集表单值 → 发送 `save_module_settings` WS 请求 → 成功后刷新 info 卡片

### 12.4 模块详情页渲染逻辑

进入模块详情页时：

1. 收到 `get_modules` 响应，找到目标模块的 `cards`
2. 按 `info → actions → settings` 顺序渲染（对应标签切换或垂直分区）
3. **info 区域**：只读展示，实时响应 `module_cards_updated` 推送
4. **actions 区域**：按钮点击触发 WS 请求，显示 loading 状态直到响应
5. **settings 区域**：可编辑，保存时发送 `save_module_settings`

### 15.4 WS 接口（模块市场）

| 方法 | 说明 |
|------|------|
| `get_modules` | 获取已加载模块列表（含 cards） |

| `uninstall_module` | 卸载模块（从模块目录移除） |
| `load_module` | 加载已安装模块 |

**安装响应**：


## 16. 架构约束与实现建议

### 16.1 跨平台一致性

各平台 UI 实现须满足以下约束：

| 约束 | 说明 |
|------|------|
| 统一的数据源 | 所有页面数据均来自 WS 响应/推送，UI 不持有硬编码模块状态 |
| cards 渲染器复用 | 同一套 CardsRenderer 实现复用于状态页、模块页、市场页 |
| 连接状态独立 | WebSocket 连接状态单独管理，不绑定任何页面生命周期 |
| 响应式布局 | 主内容区使用响应式网格/列表，适应不同屏幕宽度 |

### 16.2 性能注意事项

- WS 推送（`status_update`、`module_status_changed`）触发 UI 更新时，应使用增量更新而非全量刷新
- 模块卡片网格在模块数量超过 20 个时，应支持虚拟化滚动（virtualized list）
- `get_modules` 响应中 cards 数据可能较大，避免在每次推送时完整序列化和反序列化

### 16.3 错误处理

| 场景 | UI 行为 |
|------|--------|
| WS 连接失败 | 状态栏显示"Core 未连接"，所有页面显示骨架屏/占位符 |
| WS 请求超时（>10s） | 显示 toast 提示"请求超时，请重试" |
| `module_status_changed` 推送时模块卡片正在编辑 settings | 保持编辑状态不中断，仅更新 info 区域 |
| Core 重启 | 自动重连成功后，UI 全量刷新一次（`get_core_status` + `get_modules`） |
