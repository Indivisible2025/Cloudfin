# Cloudfin UI 技术规格

> 版本：v0.1.0  
> 日期：2026-04-03  
> 状态：**草案 — 待审查**

---

## 1. 概述

Cloudfin UI 是独立于 Core 的平台原生应用，通过 HTTP API 与 Cloudfin Core 守护进程通信。

```
┌─────────────────────────────────────┐
│         Cloudfin UI (独立进程)        │
│  Flutter / SwiftUI / Compose / WinUI  │
└──────────────┬──────────────────────┘
               │ HTTP JSON (127.0.0.1:19001)
               ▼
┌─────────────────────────────────────┐
│      Cloudfin Core (守护进程)          │
│         127.0.0.1:19001              │
└──────────────┬──────────────────────┘
               │ .so 插件
               ▼
┌─────────────────────────────────────┐
│         cloudfin-mod/               │
│   p2p / crdt / tor / i2p / storage   │
└─────────────────────────────────────┘
```

---

## 2. 技术栈

| 平台 | UI 框架 | 说明 |
|------|---------|------|
| Android | Kotlin + Jetpack Compose | 首选平台 |
| iOS | Swift + SwiftUI | 备选平台 |
| Windows | C# + WinUI 3 | 桌面平台 |
| macOS | Swift + SwiftUI | 桌面平台 |
| Linux | Rust (QML) / GTK | 桌面平台 |

**跨平台方案**：
- Flutter + Kotlin/Swift 嵌入（短期快速）
- 纯平台原生（长期维护）

## 3. 屏幕结构

### 3.1 底部导航（5 Tab）

```
┌────────────────────────────────────┐
│  [状态]  [模块]  [网络]  [同步]  [设置] │

```

每个 Tab 对应一个独立页面。

---

### 3.2 页面详情

#### 页面 1：状态（Status）

**路径**：`/` 或 `/status`

**功能**：显示 Core 和各模块运行状态

**UI 布局**：
```
┌────────────────────────────────┐
│ ☁ 云鳞                    v0.1.0│  ← 顶部栏
├────────────────────────────────┤
│  ● 运行中                      │  ← 状态卡片
│  已运行：2小时 34分             │
│  设备ID：abcd-1234             │
├────────────────────────────────┤
│  📡 P2P          ● 已连接       │  ← 模块状态列表
│  🔒 TOR          ○ 已停止       │
│  🌐 I2P          ● 已连接       │
│  🔄 同步         ● 已连接       │
├────────────────────────────────┤
│  流量：↑ 2.3 MB  ↓ 5.1 MB      │  ← 流量统计
└────────────────────────────────┘
```

**API 调用**：
- `GET /api/core/status` → 获取 Core 状态
- `GET /api/modules` → 获取模块列表

**交互**：
- 下拉刷新 → 重新请求 `/api/core/status`
- 点击模块 → 跳转 `/modules/{id}`

#### 页面 2：模块（Modules）

**路径**：`/modules`

**功能**：管理 P2P / TOR / I2P / CRDT / Storage 模块

**UI 布局**：
```
┌────────────────────────────────┐
│ 模块管理                        │
├────────────────────────────────┤
│ ┌────────────────────────────┐ │
│ │ 📡 P2P 网络                 │ │
│ │ libp2p v0.54              │ │
│ │ [●] 已连接  3 个节点       │ │
│ │ [启动] [配置] [详情]       │ │
│ └────────────────────────────┘ │
│ ┌────────────────────────────┐ │
│ │ 🔒 TOR 匿名网络             │ │
│ │ [○] 已停止                 │ │
│ │ [启动] [配置] [详情]       │ │
│ └────────────────────────────┘ │
│ ┌────────────────────────────┐ │
│ │ 🔄 CRDT 同步               │ │
│ │ [●] 运行中  2 个文档       │ │
│ │ [停止] [配置] [详情]       │ │
│ └────────────────────────────┘ │
└────────────────────────────────┘
```

**API 调用**：
- `GET /api/modules` → 模块列表
- `POST /api/modules` → 注册模块
- `DELETE /api/modules/:id` → 注销模块
- `POST /api/plugins/load` → 动态加载
- `DELETE /api/plugins/unload/:id` → 动态卸载

**交互**：
- 卡片点击 → 展开/收起模块详情
- [启动/停止] → `POST /api/plugins/load` 或 `DELETE /api/plugins/unload/:id`
- [配置] → 跳转 `/modules/{id}/config`
- [详情] → 跳转 `/modules/{id}`

## 4. UI ↔ Core 通信协议

### 4.1 连接检测

UI 启动时先检测 Core 是否可用：

```dart
Future<bool> checkCoreAvailable() async {
  try {
    final resp = await http.get(Uri.parse('http://127.0.0.1:19001/api/core/status'));
    return resp.statusCode == 200;
  } catch (e) {
    return false;
  }
}
```

### 4.2 错误处理

| 情况 | UI 行为 |
|------|---------|
| Core 未启动 | 弹窗："Core 未运行，是否启动？" |
| API 超时（5s） | Toast："连接超时" |
| API 错误 | Toast：显示 error 字段 |
| 网络错误 | 显示离线状态，自动重试 |

### 4.3 实时状态（WebSocket）

Core 定时推送状态更新（可选）：

```
WebSocket: ws://127.0.0.1:19001/ws/status
消息格式：{ "type": "status_update", "data": { ... } }
```

## 6. 状态管理

采用 **BLoC 模式**（Flutter）：

```
┌──────────────┐    ┌──────────────┐
│   UI Layer   │◄──►│   BLoC Layer  │
│  (Widgets)   │    │ (状态管理)    │
└──────────────┘    └───────┬──────┘
                             │
                      ┌──────▼──────┐
                      │  Repository │
                      │ (API 调用)   │
                      └──────┬──────┘
                             │
                      ┌──────▼──────┐
                      │  Core HTTP  │
                      │   API        │
                      └─────────────┘
```

**BLoC 列表**：
- `CoreStatusBloc` → Core 状态
- `ModulesBloc` → 模块列表和操作
- `NetworkBloc` → P2P 网络状态
- `SyncBloc` → CRDT 同步状态
- `SettingsBloc` → 配置管理

## 7. 安装引导流程

首次启动 UI 时引导用户完成安装：

```
步骤 1：欢迎
  → "欢迎使用云鳞"
  → [开始安装]

步骤 2：检测 Core
  → 检测 /opt/cloudfin/cloudfin-core 是否存在
  → 不存在 → 提示下载

步骤 3：连接 Core
  → 测试 127.0.0.1:19001 连通性
  → 成功 → 进入主页
  → 失败 → 显示错误

步骤 4：模块安装
  → 显示可选模块列表
  → 用户选择要安装的模块
  → 安装完成
```

## 8. 视觉设计

### 8.1 主题

支持 **暗色 / 亮色 / 跟随系统** 三种模式。

**暗色主题色板**：
- 主色：`#6C63FF`（紫蓝）
- 背景：`#121212`
- 卡片：`#1E1E1E`
- 文字：`#E0E0E0`
- 成功：`#4CAF50`
- 警告：`#FFC107`
- 错误：`#F44336`

### 8.2 圆角 & 间距

- 卡片圆角：16px
- 按钮圆角：12px
- 内边距：16px
- 元素间距：12px

### 8.3 图标

使用 Material Icons 或 Font Awesome，模块对应图标：
- P2P：`wifi_tethering`
- TOR：`vpn_key`
- I2P：`lan`
- CRDT：`sync`
- Storage：`storage`
- 设置：`settings`
