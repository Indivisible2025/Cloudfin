## 1. 概述

### 1.1 什么是安装引导流程

安装引导流程 = **用户首次使用 App 时，UI 引导用户完成 Core 和模块的安装配置**。

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **简洁** | 步骤少、信息清晰、用户不困惑 |
| **自动化** | 自动检测环境、自动配置，用户只需确认 |
| **容错** | 安装失败有明确提示和重试机制 |
| **可跳过** | 高级用户可跳过非必要步骤 |
| **进度可见** | 每一步都有明确状态，用户知道发生了什么 |

### 1.3 安装类型

| 类型 | 说明 | 用户操作 |
|------|------|---------|
| **首次安装** | 全新安装，所有组件都要安装 | 按引导一步步来 |
| **更新安装** | 已有旧版本，升级到新版本 | 自动检测，一键更新 |
| **模块安装** | 安装或更新某个模块 | 从模块市场安装 |

## 2. 安装流程总览

### 2.1 首次安装流程

```
┌─────────────────────────────────────────────────────────────┐
│                    首次安装引导                              │
│                                                              │
│  第1步：欢迎页 ──▶ 第2步：权限 ──▶ 第3步：Core安装 ──▶       │
│                                                              │
│  第4步：模块选择 ──▶ 第5步：模块安装 ──▶ 第6步：完成 ──▶       │
│                                                              │
│  ──▶ 启动 Cloudfin                                           │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 详细步骤

| 步骤 | 名称 | 说明 |
|------|------|------|
| 1 | 欢迎页 | 介绍 Cloudfin，让用户了解这是什么 |
| 2 | 权限请求 | 请求必要的权限（存储、网络等）|
| 3 | Core 安装 | 下载/安装 Cloudfin Core |
| 4 | 模块选择 | 让用户选择需要的功能模块 |
| 5 | 模块安装 | 下载/安装选中的模块 |
| 6 | 完成确认 | 确认所有组件正常运行 |
| 7 | 启动 App | 进入主界面 |

## 4. 第二步：权限请求

### 4.1 需要请求的权限

| 权限 | 用途 | 重要性 |
|------|------|-------|
| **存储权限** | 读写同步文件、安装模块 | 必须 |
| **网络权限** | Core 和模块通信、模块间通信 | 必须 |
| **后台运行权限** | 保持 Core 在后台运行 | 强烈建议 |
| **通知权限** | 推送同步状态、连接状态 | 建议 |

```
┌─────────────────────────────────────────┐

│                                          │
│    [✓] 网络权限                         │
│        用于模块间通信                     │
│                                          │
│    [ ] 后台运行权限  ⭐建议              │
│        保持 Cloudfin 后台运行             │
│                                          │
│    [ ] 通知权限                          │
│        推送同步和连接状态                  │
```

### 4.3 逻辑

- 必须权限（存储、网络）用户拒绝 → 弹窗说明必须授权才能继续
- 可选权限用户拒绝 → 记录偏好，继续下一步
- 点击"继续" → 申请权限 → 进入第三步
- 点击"跳过" → 记录偏好（影响功能提示用户）→ 进入第三步

## 5. 第三步：Core 安装

### 5.1 检测逻辑

```
App 启动时检测：
    │
    ├─ Core 已安装？
    │   ├─ 是：Core 版本 = 最新版本？
    │   │   ├─ 是：✅ Core 就绪 → 进入第四步
    │   │   └─ 否：🔄 有更新 → 显示更新弹窗 → 用户确认 → 更新
    │   │
    │   └─ 否：❌ Core 未安装 → 进入安装流程
    │
    └─ Core 运行中？
        ├─ 是：✅ Core 运行正常
        └─ 否：🔄 Core 未运行 → 自动启动 → 失败 → 显示错误
```

### 5.3 安装方式

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| **自动下载** | 从 GitHub Release 下载 Core | 网络正常 |
| **本地安装** | 用户手动选择 APK / 二进制文件 | 网络受限 |
| **内置** | Core 随 App 一起打包（Android APK）| APK 安装时已包含 |

### 5.4 安装后验证

```
Core 安装完成后：
    │
    ├─ 启动 Core（后台进程）
    │
    ├─ 等待 Core 就绪（最多 10 秒）
    │   ├─ 成功：✅ Core 运行中 → 进入第四步
    │   └─ 失败：❌ Core 启动失败 → 显示错误日志 → 重试或手动排查
    │
    └─ 验证 WebSocket 连接
        └─ 尝试连接 ws://localhost:19001/ws
            ├─ 成功：✅ 连接正常
            └─ 失败：❌ 连接失败 → 重试
```

### 7.2 安装状态

| 状态 | 显示 | 说明 |
|------|------|------|
| ⏳ 等待中 | 灰色 | 等待其他模块安装完成 |
| 🔄 正在安装 | 蓝色进度条 | 正在下载/安装 |
| ✓ 已安装 | 绿色勾 | 安装成功 |
| ✗ 安装失败 | 红色叉 | 安装失败，可重试 |

### 7.3 批量安装

- 所有选中模块**同时安装**（并行下载）
- 每个模块独立进度
- 一个失败不影响其他
- 失败后显示错误和重试按钮

### 7.4 安装来源

| 来源 | 说明 | 优先级 |
|------|------|-------|
| GitHub Release | 官方编译的模块文件 | 优先 |
| 模块市场 | 第三方模块 | 次优先 |
| 本地文件 | 用户手动选择 | 最后 |

### 7.5 安装后验证

```
每个模块安装完成后：
    │
    ├─ 加载模块：modules.load { moduleId: "p2p" }
    │
    ├─ 验证模块运行：modules.list
    │   ├─ 模块在列表中且状态=loaded/running → ✅
    │   └─ 失败 → ❌ 显示错误
    │
    └─ 记录已安装模块到配置
```

## 9. 第七步：启动主界面

### 9.1 启动流程

```
点击"启动 Cloudfin"
    │
    ├─ Core 未运行？
    │   └─ Core 已停止 → 自动启动 Core
    │
    ├─ 等待 Core 就绪（最多 5 秒）
    │
    ├─ 建立 WebSocket 连接
    │   └─ 连接 ws://localhost:19001/ws
    │
    ├─ 发送 HELLO + AUTH
    │
    ├─ 验证连接成功
    │
    └─ 进入主界面
```

### 9.2 主界面加载

```
进入主界面后：
    │
    ├─ 获取 Core 状态：core.status
    │
    ├─ 获取模块列表：modules.list
    │
    ├─ 显示各模块状态（P2P/Tor/I2P/CRDT/Crypto）
    │
    └─ 界面就绪，用户可操作
```

## 10. 更新安装流程

### 10.1 检测更新

```
App 启动时检测更新：
    │
    ├─ 读取本地 Core 版本
    │
    ├─ 查询 GitHub 最新 Release
    │   ├─ 有新版本 → 显示更新弹窗
    │   └─ 无新版本 → 正常启动
    │
    ├─ 读取本地模块版本
    │
    └─ 对比模块最新版本
        ├─ 有模块更新 → 显示更新弹窗
        └─ 全部最新 → 正常启动
```

```

    │   └─ 用户未选中 → 跳过
```

```
    │
    ├─ 导入方式：
    │   ├─ 从文件导入
    │   ├─ 扫描二维码
    │   └─ 从其他设备导入
    │
    ├─ 验证配置
    │   ├─ 版本兼容性检查
    │   └─ 模块完整性检查
    │
    ├─ 确认导入
    │   └─ 显示将导入的内容
    │
    └─ 执行导入
        ├─ 覆盖/合并选择
        └─ 重启服务
```

## 13. 错误处理

### 13.1 常见错误及解决方案

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| 下载失败 | 网络问题 | 切换网络 / 重试 / 手动安装 |
| 安装失败 | 磁盘空间不足 | 清理空间后重试 |
| Core 启动失败 | 端口被占用 | 结束占用端口的进程 |
| 模块加载失败 | 缺少依赖 | 安装依赖模块 |
| 签名验证失败 | 文件被篡改 | 重新下载 |
| 权限不足 | 未授权 | 去设置授权 |

### 13.2 错误日志

```
所有安装错误都记录到日志：
    │
    └─ 日志内容：
        ├─ 错误类型
        ├─ 错误信息
        ├─ 发生时间
        ├─ 设备信息
        ├─ App 版本
        └─ Core 版本

用户可导出日志用于排查问题
```
