# Cloudfin 桌面端 UI 规格

> 版本：v0.1.0-draft  
> 日期：2026-04-03  
> 状态：**草案 — 待完善**  
> 依赖：SPEC-UI-Common.md（跨平台通用设计）

---

## 1. 概述

本文档定义 **桌面端平台**（Windows / macOS / Linux）的 UI 适配规范，补充 SPEC-UI-Common.md 中的通用设计。

桌面端采用**窗口化设计**，与移动端的全屏页面有所不同。

---

## 2. 技术栈

| 平台 | UI 框架 | 语言 |
|------|---------|------|
| **Windows** | WinUI 3 / C++ Qt | C# / C++ |
| **macOS** | SwiftUI / AppKit | Swift / Objective-C |
| **Linux** | Qt / GTK / Rust QML | C++ / Rust |

> **注意**：Linux 发行版众多，UI 实现需考虑主流发行版（Ubuntu/Debian、Fedora、Arch）的差异。

---

## 3. 桌面端通用布局

### 3.1 主窗口布局

```
┌─────────────────────────────────────────────────────────┐
│  [≡]  Cloudfin                         [_][□][×]     │  ← 标题栏（系统/自定义）
├────────┬──────────────────────────────────────────────┤
│        │                                              │
│  P2P   │                                              │
│  Tor   │              内容区域                         │
│  I2P   │         （模块状态和控制）                   │
│  同步   │                                              │
│        │                                              │
│────────│                                              │
│  设置   │                                              │
│  关于   │                                              │
│        │                                              │
├────────┴──────────────────────────────────────────────┤
│  状态栏：Core 运行中 | P2P: 3 节点 | 上: 1.2MB/s     │  ← 状态栏
└─────────────────────────────────────────────────────┘
     ↑                ↑
   侧边栏          内容区
```

### 3.2 窗口尺寸

| 类型 | 尺寸 | 说明 |
|------|------|------|
| **最小尺寸** | 800 x 600 | 保证可用性 |
| **默认尺寸** | 1200 x 800 | 最佳体验 |
| **最大尺寸** | 无限制 | 全屏显示 |

### 3.3 窗口行为

| 行为 | 说明 |
|------|------|
| 最小化到托盘 | 关闭按钮默认最小化到系统托盘 |
| 托盘菜单 | 右键显示快捷操作 |
| 启动时隐藏 | 可选，开机自启时最小化到托盘 |

## 4. 系统托盘

| 平台 | 实现 |
|------|------|
| **Windows** | System Tray (NotifyIcon) |
| **macOS** | Menu Bar Extra / NSStatusItem |
| **Linux** | libappindicator (GTK) / Qt System Tray |

**托盘菜单项**：
```
┌─────────────────────┐
│  🌙 Cloudfin         │
├─────────────────────┤
│  [P2P]  ● 运行中    │
│  [Tor]  ○ 已停止    │
│  [I2P]  ○ 已停止    │
├─────────────────────┤
│  打开主窗口          │
│  设置                │
│  关于                │
├─────────────────────┤
│  退出                │
└─────────────────────┘
```

## 5. 平台特定限制

### 5.1 Windows

| 限制 | 说明 |
|------|------|
| UAC | 可能需要管理员权限安装 |
| Windows Defender | 首次运行可能被误报 |
| 多显示器 | 支持多窗口、多显示器 |
| HDR | 需适配高动态范围显示 |

### 5.2 macOS

| 限制 | 说明 |
|------|------|
| App Sandbox | 应用沙盒限制文件访问 |
| Notarization | 发布前需要公证 |
| Menu Bar | 可选使用 NSStatusItem |
| Retina | 原生支持高清显示 |

### 5.3 Linux

| 限制 | 说明 |
|------|------|
| 发行版差异 | Ubuntu/Debian vs Fedora vs Arch 行为不同 |
| 桌面环境 | GNOME / KDE / XFCE / GTK / Qt 渲染差异 |
| 系统托盘 | 各桌面环境实现不同 |
| 打包格式 | DEB / RPM / AppImage / Snap |

## 6. 与 SPEC-UI-Common.md 的差异

### 6.1 导航

| 通用设计 | 桌面端适配 |
|---------|----------|
| 底部 Tab Bar | 左侧边栏 NavigationRail |
| — | 可选顶部菜单栏（macOS Menu Bar）|

### 6.2 窗口管理

| 功能 | 说明 |
|------|------|
| 最小化 | 最小化到窗口/托盘 |
| 最大化 | 全屏 / 最大化切换 |
| 调整大小 | 支持拖拽调整 |
| 多窗口 | 设置页面可弹出新窗口 |

### 6.3 键盘快捷键

| 快捷键 | 操作 |
|--------|------|
| `Ctrl/Cmd + Q` | 退出 |
| `Ctrl/Cmd + W` | 关闭窗口/最小化到托盘 |
| `Ctrl/Cmd + ,` | 打开设置 |
| `Ctrl/Cmd + R` | 刷新 |
| `Ctrl/Cmd + 1-5` | 切换 Tab |
| `F11` | 全屏切换 |

## 7. 托盘与后台运行

### 7.1 托盘图标状态

| 状态 | 图标 | 说明 |
|------|------|------|
| 运行中 | 绿色圆点 | Core 运行正常 |
| 部分连接 | 黄色圆点 | 部分模块连接 |
| 停止 | 灰色圆点 | 全部停止 |
| 错误 | 红色圆点 | 有模块出错 |

### 7.2 托盘气泡通知

| 场景 | 通知内容 |
|------|---------|
| 模块连接成功 | "P2P 已连接，3 个节点" |
| 模块断开 | "Tor 连接已断开" |
| 同步完成 | "同步完成，5 个文件" |
| 发现新节点 | "发现新节点：xxx" |
