## 1. 概述

### 1.1 文档目的

本文件定义**所有平台通用的 UI 设计规范**，包括：
- 通用交互逻辑
- 通用页面结构
- 通用组件样式
- 通用导航模式

各平台文档（SPEC-UI-Android.md、SPEC-UI-iOS.md、SPEC-UI-Desktop.md）基于本文档，补充平台特定的适配和限制。

### 1.2 平台差异概述

| 平台 | UI 框架 | 交互特点 | 主要限制 |
|------|---------|----------|---------|
| **Android** | Jetpack Compose | 手势操作、返回键、底部导航 | 系统权限弹窗、后台限制 |
| **iOS** | SwiftUI | 手势操作、Home 键、Tab Bar | 沙盒限制、后台冻结、权限申请更严格 |
| **Windows** | WinUI3 / Qt | 鼠标+键盘、窗口操作 | 窗口管理、多显示器支持 |
| **macOS** | SwiftUI | 触控板+鼠标、Menu Bar | App Store 审核、沙盒限制 |
| **Linux** | GTK / QML | 鼠标+键盘、窗口操作 | 发行版差异（Debian/Ubuntu vs Fedora vs Arch）|

## 2. 通用页面结构

### 2.1 主界面架构

所有平台的主界面采用**底部导航 + 内容区**的通用布局：

```
┌─────────────────────────────────────────┐
│  状态栏（系统提供）                        │
├─────────────────────────────────────────┤
│                                          │
│                                          │
│              内容区域                      │
│         （各 Tab 的页面内容）              │
│                                          │
│                                          │
├─────────────────────────────────────────┤
│  [P2P] [Tor] [I2P] [同步] [设置]        │  ← 底部导航栏
└─────────────────────────────────────────┘
```

**说明**：
- 5 个 Tab：P2P、Tor、I2P、同步、设置
- 底部导航栏固定，内容区域可滚动
- 各平台可根据规范调整图标和文案

### 2.2 状态栏

| 元素 | 位置 | 说明 |
|------|------|------|
| 连接状态图标 | 左上 | Core 连接状态（已连接/未连接/连接中）|
| 云鳞 Logo | 中上 | 可选，部分平台隐藏 |
| 更多按钮 | 右上 | 快捷菜单（通知、帮助、关于）|

### 2.3 页面通用结构

```
页面
├── 页面标题栏
│   ├── 返回按钮（次级页面）
│   ├── 页面标题
│   └── 操作按钮（可选）
├── 内容区
│   ├── 状态卡片
│   ├── 操作区
│   └── 详情区
└── 底部操作栏（可选）
```

## 3. 通用组件

### 3.1 状态卡片

**用途**：显示模块状态（运行中/已停止/未安装）

**通用结构**：
```
┌─────────────────────────────────────────┐
│  [图标]  模块名称              [状态指示]  │
│          简短描述                         │
│          [主要操作按钮]                    │
└─────────────────────────────────────────┘
```

**状态指示**：
| 状态 | 颜色 | 说明 |
|------|------|------|
| `running` | 绿色 | 运行中 |
| `stopped` | 灰色 | 已停止 |
| `error` | 红色 | 错误 |
| `loading` | 蓝色 | 加载中 |
| `not_installed` | 橙色 | 未安装 |

### 3.2 操作按钮

| 类型 | 样式 | 用途 |
|------|------|------|
| **主操作** | 填充色、主色 | 连接、启动、发送 |
| **次操作** | 描边样式 | 设置、更多 |
| **危险操作** | 红色填充 | 断开、删除 |
| **禁用状态** | 灰色，不可点击 | 条件不满足时 |

### 3.3 输入控件

**通用输入类型**：
| 控件 | 平台通用性 | 说明 |
|------|-----------|------|
| 文本输入框 | ✅ | 单行/多行文本 |
| 数字输入 | ✅ | 端口号等 |
| 开关 | ✅ | 启用/禁用选项 |
| 选择器 | ⚠️ | 平台差异较大，需适配 |
| 日期时间 | ⚠️ | 平台差异较大，需适配 |

### 3.4 对话框

| 类型 | 用途 |
|------|------|
| **确认对话框** | 删除、断开等危险操作 |
| **输入对话框** | 输入地址、端口等 |
| **加载对话框** | 等待异步操作完成 |
| **底部弹窗** | 更多选项、排序等 |

## 4. 通用交互逻辑

### 4.1 连接流程

```
用户点击"连接"
    │
    ▼
显示加载状态（按钮禁用，显示 spinner）
    │
    ▼
调用 Core API：modules.load { moduleId: "xxx" }
    │
    ├── 成功 → 更新卡片状态为 running → 显示成功提示
    │
    └── 失败 → 显示错误对话框（错误码 + 错误信息）
                  ├─ 重试按钮
                  └─ 取消按钮
```

### 4.2 断开流程

```
用户点击"断开"
    │
    ▼
显示确认对话框："断开后当前连接将中断"
    │
    ├── 用户点"取消" → 无操作
    │
    └── 用户点"确认" → 显示加载状态 → 调用 modules.unload
                       │
                       ├── 成功 → 更新状态为 stopped
                       └── 失败 → 显示错误
```

### 4.3 刷新流程

```
用户下拉刷新 / 点击刷新按钮
    │
    ▼
显示刷新指示器
    │
    ▼
调用 core.status 获取最新状态
    │
    ▼
更新所有模块卡片状态
    │
    ▼
隐藏刷新指示器
```

### 4.4 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| **网络错误** | 重试按钮 + "检查网络连接"提示 |
| **超时错误** | 重试按钮 + "请求超时"提示 |
| **权限错误** | 跳转设置页面 + "需要 XXX 权限"提示 |
| **Core 未运行** | "Core 未运行"提示 + "启动"按钮 |
| **模块加载失败** | 显示具体错误码 + 重试按钮 |
